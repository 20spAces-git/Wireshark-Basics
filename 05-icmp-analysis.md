# Lab 05 - ICMP Analysis

## Objective

Use Wireshark to identify and analyze ICMP traffic generated during network connectivity testing.

---

## Lab Environment

### Operating System

- Windows 11

### Tools Used

- Wireshark 4.6.6
- Npcap 1.88

### Network Interface

- Wi-Fi Adapter

---

## Procedure

### Step 1 - Start Packet Capture

Opened Wireshark and started capturing traffic on the active Wi-Fi adapter.

### Step 2 - Generate ICMP Traffic

Opened Command Prompt and executed:

ping google.com

### Step 3 - Stop Packet Capture

Stopped the capture after the ping test completed.

### Step 4 - Apply ICMP Filter

Used the following display filter:

icmp

### Step 5 - Analyze ICMP Packets

Reviewed ICMP Echo Requests and Echo Replies generated by the ping command.

## Key Concepts

### What is ICMP?

ICMP (Internet Control Message Protocol) is used for diagnostics and error reporting on IP networks.

Common uses include:

- Ping
- Connectivity Testing
- Network Troubleshooting

---

## Observations

Observed:

- ICMP Echo Request Packets
- ICMP Echo Reply Packets

Example Flow:

Local Computer
↓

ICMP Echo Request

↓

Remote Host

↓

ICMP Echo Reply

↓

Local Computer

This exchange confirms successful communication between systems.

## What I Learned

ICMP is commonly used to test connectivity between devices.

The ping command generates ICMP traffic that can be captured and analyzed using Wireshark.

By examining ICMP packets, analysts can verify communication and troubleshoot network issues.

---

## Cybersecurity Relevance

ICMP analysis can help:

- Verify network connectivity
- Troubleshoot communication problems
- Investigate network behavior
- Support incident response activities
- Assist with forensic investigations

## Safety Considerations
