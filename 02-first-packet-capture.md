# Lab 02 - First Packet Capture

## Objective

Capture live network traffic using Wireshark and identify common protocols observed during normal web browsing activity.

---

## Lab Environment

### Operating System

- Windows 11

### Tools Used

- Wireshark 4.6.6
- Npcap 1.88

### Network Interface

- Wi-Fi Adapter

---

## Procedure

### Step 1 - Open Wireshark

Launched Wireshark and identified the active Wi-Fi network interface.

### Step 2 - Start Packet Capture

Selected the active Wi-Fi adapter and began capturing live network traffic.

### Step 3 - Generate Network Activity

Generated network traffic by visiting websites and interacting with online services.

Examples:

- Google
- GitHub
- TryHackMe

### Step 4 - Stop Packet Capture

Stopped the capture after collecting network traffic.

### Step 5 - Analyze Packets

Reviewed packet information and examined:

- Source IP Addresses
- Destination IP Addresses
- Protocols
- Ports
- Packet Details

### Step 6 - Apply Filters

Applied protocol filters to isolate specific traffic types.

Example Filters:

dns
tcp
udp

## What I Learned

Packet captures contain a large amount of network activity generated by applications, web browsers, and operating system services.

Wireshark allows analysts to filter traffic and focus on specific protocols of interest.

Filtering traffic makes packet analysis significantly easier and is a fundamental skill for cybersecurity and DFIR investigations.

---

## Cybersecurity Relevance

Packet analysis helps security professionals:

- Investigate suspicious network activity
- Identify communication patterns
- Troubleshoot connectivity issues
- Analyze incidents
- Support digital forensic investigations

---

## Safety Considerations

- Only capture traffic on systems and networks you own or are authorized to analyze.
- Do not inspect or collect network traffic belonging to other users without permission.
- Follow applicable security and privacy policies.
