# Lessons Learned - Wireshark Basics

## Overview

This repository documents my introduction to Wireshark and packet analysis. Through a series of hands-on labs, I learned how to capture, filter, and analyze network traffic while developing foundational skills relevant to cybersecurity and Digital Forensics & Incident Response (DFIR).

## Key Skills Gained

### Packet Capture

Learned how to:

* Identify active network interfaces
* Start and stop packet captures
* Generate network traffic for analysis
* Save packet capture files

### Traffic Analysis

Learned how to:

* Examine source and destination IP addresses
* Identify protocols in use
* Review packet details
* Analyze network communications

### Protocol Identification

Gained experience identifying:

* DNS
* ARP
* ICMP
* TCP

and understanding their role in network communications.

## What Surprised Me

### Amount of Network Traffic

Even when a computer appears idle, there is often a significant amount of network activity occurring in the background.

### Importance of DNS

Many Internet activities depend on DNS before communication can occur.

### TCP Activity

Most web browsing traffic relies heavily on TCP and secure communications.
